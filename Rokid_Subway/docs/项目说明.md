# Rokid AR 眼镜 — 地铁导航 App

## 项目概述

为 Rokid AR 智能眼镜开发的**离线语音驱动地铁导航**应用。用户通过语音说出起点和终点（如"从天安门到中关村"），系统自动识别语音、查询地铁路线、显示详细换乘方案。

## 核心技术栈

| 技术 | 用途 |
|------|------|
| Kotlin | Android 应用开发 |
| Sherpa-onnx (Zipformer) | 离线语音识别（ASR），中文 14M 参数模型 |
| 高德地图 Web API | 地理编码 + POI搜索 + 公交路线规划 |
| OkHttp + Gson | 网络请求 + JSON 解析 |
| Android AudioRecord | 麦克风音频采集（AudioSource.MIC） |
| Kotlin Coroutines | 异步操作 |

## 功能特性

1. **离线语音识别** — Sherpa-onnx Zipformer 模型，无需联网即可识别中文语音
2. **智能语义解析** — 支持 "从A到B"、"去A"、"A到B" 等多种说法
3. **跨城市查询** — 人在武汉也能查北京地铁，自动从地名推断城市
4. **多方案对比** — 显示多条路线方案，包含时间、费用、换乘次数
5. **详细换乘信息** — 每段路线的上下车站、途经站、步行距离
6. **AR 眼镜适配** — 480x640 竖屏、绿色 HUD 风格、触控板前后滑动操作

## Rokid 眼镜适配要点

- **AudioSource.MIC(1)** — Rokid 定制 HAL 不支持 VOICE_RECOGNITION(6)
- **DPAD_RIGHT/LEFT** — 触控板前后滑动映射的键码
- **无键盘** — 纯语音交互，无文字输入
- **窗口标志 6815872** — 确保在眼镜 HUD 模式正确显示

## 启动方式

```bash
# 语音输入模式（无需任何参数）
adb shell am start -n com.rokid.transit/.ui.TransitMainActivity

# 指定出发地（经纬度）
adb shell am start -n com.rokid.transit/.ui.TransitMainActivity \
  --es origin "116.397428,39.90923" --es city "北京市"

# 直接指定起终点名称
adb shell am start -n com.rokid.transit/.ui.TransitMainActivity \
  --es origin_name "天安门" --es destination "中关村"
```
