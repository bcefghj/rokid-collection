## 一、应用定位与本质

「智慧生活」在应用列表里显示的名称是 **"地图"**（`APPName=地图`），实质是雷鸟X3 Pro上的 **AR导航应用**，不是一个综合性的"生活服务"应用。

核心功能：
- **AR叠加导航**：把3D箭头、路线信息叠加到眼前的真实世界
- **语音搜索目的地**：单击镜腿触发，全程语音交互
- **附近地点探索**：首页展示面前的地点名称卡片
- **步行/骑行路线规划**：对接高德地图引擎

## 二、技术架构

### 2.1 引擎层：Unity + OpenXR

这个应用**不是用原生Android UI写的**，而是用 **Unity游戏引擎** 开发的AR应用。这是 AR 类应用的主流选择。

```
Unity引擎架构
├── Unity版本：定制版（支持Android 12 / targetSdk 31）
├── 渲染方式：boot.config 中 gfx-disable-mt-rendering=1（禁用多线程渲染）
├── XR框架：Unity OpenXR 1.7.0
├── 输入：OpenXR Input（统一XR输入抽象）
├── 显示：OpenXR Display（双目显示管理）
└── 脚本运行时：IL2CPP（AOT编译，比Mono更快更难逆向）
```

IL2CPP 的意思是：C#代码被提前编译成C++，再编译成ARM64机器码，存在 `libil2cpp.so`（62MB，这是整个APK里最大的文件）。因此**无法用jadx直接反编译出可读C#代码**，需要专用工具（如Il2CppDumper）。

### 2.2 主要Native库（lib/arm64-v8a/）

| 库文件 | 大小 | 功能 |
|--------|------|------|
| `libil2cpp.so` | 62MB | IL2CPP运行时 + 全部业务逻辑（C#编译产物） |
| `libAMapSDK_NAVI_v10_1_302.so` | 60MB | 高德地图导航引擎（版本10.1.302） |
| `libunity.so` | 25MB | Unity引擎核心 |
| `libAMapSDK_AGENT_v1_0_0.so` | 2MB | 高德地图Agent（数据/网络层） |
| `libRayNeoXRSDKClient.so` | 418KB | 雷鸟XR SDK客户端 |
| `libRayNeoXRUnityInterfaces.so` | 243KB | Unity-XR桥接接口 |
| `libUnityOpenXR.so` | 1.2MB | Unity OpenXR插件 |
| `libRayNeoXRLocalLoader.so` | 341KB | XR本地加载器 |
| `libRayNeoXRApiLayerClient.so` | 38KB | XR API层客户端 |
| `libneonui_shared.so` | 4.5MB | NeonUI界面库（雷鸟UI框架） |
| `libopenxr_loader.so` | 21KB | OpenXR标准加载器 |

**关键发现**：高德导航库（`libAMapSDK_NAVI`）有60MB，几乎和Unity引擎一样大，说明地图/导航是核心功能，不是简单嵌入。

### 4.4 磁力计校准流程
导航前需要校准磁力计（确保方向准确）：
```
1. 弹出校准提示 → "眼镜磁力计发生异常，可能导致方向偏差"
2. "请保持头部水平朝前，校准过程中请勿转动身体"
3. "请大幅度左右转头" → "请持续向右转头" → "请持续向左转头"
4. "请将头摆正"
5. "请大幅度上下转头" → 上下转头引导
6. "做得很好，校准成功" ✅
```
这对应 `MMC56X3X` 磁力计传感器（系统传感器分析中已记录）。

### 4.5 手机App联动
```
定位问题提示：
  "无GPS信号，请打开手机'雷鸟AR眼镜'App"
  "请为手机App设置'始终允许'获取'精确定位'"
  "为保障地图定位，请打开手机App并授权'精确定位'"
```
说明眼镜的GPS精度较差，实际定位需要依赖手机App通过蓝牙传递GPS数据（`BtDisconnect_Title` 提示）。

### 4.6 掌上地图
```
相机权限申请原因："用于识别手掌信息，实现掌上地图功能"
未授权提示："您未授权相机权限，无法使用掌上地图"
```
**掌上地图**是一个特色功能：用摄像头识别手掌，在手掌上显示地图（AR叠加）。由于CAMERA权限未授权，此功能当前不可用。

### 4.7 HERE地图服务
```
条款说明："此地图应用需要访问设备的位置，
          并可能收集附近Wi-Fi和蜂窝信号的信息，
          以帮助改进应用所使用的HERE服务"
```
**HERE**（赫尔地图）是诺基亚旗下的地图服务商，主要服务欧美市场。这说明雷鸟X3 Pro同时集成了**高德地图（中国）** 和 **HERE地图（海外）**，是国际化版本的配置。

## 七、AR导航技术实现推断

### 7.1 坐标系融合
AR导航需要把地图坐标（经纬度）映射到眼镜视野坐标，链路：
```
GPS/蓝牙手机定位 → 高德SDK坐标变换 → Unity世界坐标 → OpenXR投影矩阵 → 双目显示
```

### 7.2 方向对齐
导航箭头指向正确需要：
```
磁力计(MMC56X3X) + 陀螺仪(LSM6DSR) → 传感器融合 → 头部朝向
                                     ↓
                              OpenXR头部追踪(MonitorService)
                                     ↓
                              地图导航方向对齐
```
这就是为什么磁力计异常时必须先校准才能导航。

### 7.3 掌上地图实现
```
摄像头 → 手掌检测（`libimage-processor.so`，5KB，轻量ML推理）
       → 手掌位置/朝向 → Unity空间锚点 → 在手掌平面渲染地图
```

### 7.4 AR箭头3D渲染
```
资源：assets/aa/Android/arrow3d_assets_all_*.bundle（76KB，轻量3D模型）
     assets/aa/Android/laserdotview_assets_all_*.bundle（10KB，激光点）
     assets/aa/Android/navigationarrowview_assets_all_*.bundle（10KB）
渲染：Unity渲染管线 → OpenXR → MicroLED双目显示（640×480，60Hz）
```

## 八、关键发现汇总

| 发现 | 详情 |
|------|------|
| **不是智慧生活，是AR地图** | 实际功能是步行/骑行AR导航，不含天气、家电控制等 |
| **Unity引擎开发** | 用游戏引擎做AR应用，IL2CPP编译，代码难以逆向 |
| **双地图引擎** | 高德（中国）+ HERE（国际），是国际化版本 |
| **GPS依赖手机** | 眼镜GPS精度弱，实际定位靠蓝牙连手机App传递 |
| **麦克风未授权** | 语音搜索功能当前无法使用（权限弹窗需用户手动允许） |
| **掌上地图** | 相机识别手掌，AR叠加地图——但CAMERA权限未授权 |
| **广告/内购SDK** | 预埋了Unity广告和Google Play内购，目前未启用 |
| **AIOS适配器** | 通过 `RayNeo.UnityAIOSAdapter.dll` 接入雷鸟AI语音OS |
| **磁力计强依赖** | 导航方向依赖磁力计，异常时必须校准才可使用 |
| **HarmonyOS字体** | 使用华为鸿蒙系统字体（HarmonyOS Sans SC），体现与华为生态的关联 |
