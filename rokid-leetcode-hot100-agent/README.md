# 💻 LeetCode Hot100 解题师 · Rokid 灵珠智能体

> 基于 Rokid AR 眼镜的算法学习智能体，通过语音讲解 LeetCode Hot100 全部 100 道经典题目的思路与解法。

## 项目简介

刷 LeetCode 不一定要盯着屏幕。本智能体让你在通勤、跑步、做家务的时候，用语音向 AI 请教任意一道 Hot100 题目的解题思路，AI 按需渐进式展开：先说题意 → 再说思路 → 用户追问才给代码。

## 功能特性

- 📖 **100 道完整覆盖**：哈希、双指针、滑动窗口、链表、二叉树、动态规划等 14 大类型
- 🗣️ **语音优先设计**：每次回复控制在 80 字以内，适配眼镜 TTS 语音播报
- 🧩 **渐进式讲解**：题意 → 核心思路 → 代码，不一次全说，减少信息量过载
- 🔄 **追问支持**：支持"为什么用这个方法""时间复杂度多少""给我看代码"等追问
- 🐍 **Python 解法**：所有题目提供 Python 完整代码

## 快速开始

### 使用方式（灵珠平台）

1. 登录 [灵珠开发者平台](https://agent-develop.rokid.com)
2. 创建知识库，上传 `LeetCode_Hot100_题解库.md`（按 `---` 分割，每段约 300 字）
3. 新建对话型智能体，名称填"LeetCode 解题师"
4. 将 `Prompt_系统提示词.md` 中的 Prompt 粘贴到系统提示词
5. 关联知识库（混合检索，召回 3 段），发布上线

### 语音交互示例

```
用户：讲一下两数之和
AI：两数之和是第1题，给你一个数组和目标值，找两个数加起来等于目标值，返回下标。核心用哈希表，一次遍历搞定，时间复杂度 O(n)。

用户：为什么用哈希表？
AI：因为要快速查找"目标值减去当前数"在不在数组里。用哈希表查找是 O(1)，如果用双重循环是 O(n²)，差太多了。

用户：给我看代码
AI：
hashmap = {}
for i, num in enumerate(nums):
    if target - num in hashmap:
        return [i, hashmap[target - num]]
    hashmap[num] = i
```

## 题解库说明

## Prompt 设计亮点

| 特性 | 说明 |
|------|------|
| 渐进式展开 | 先题意再思路，不主动给代码，减少信息过载 |
| 追问驱动 | 用户每次追问才展开下一层，符合 AR 眼镜交互节奏 |
| 80 字限制 | 语音播报约 25 秒，最佳听觉体验区间 |
| 多种提问方式 | 支持题号、题名、类型三种查询方式 |
