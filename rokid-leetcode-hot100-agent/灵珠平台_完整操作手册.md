# LeetCode Hot100 解题师 · 灵珠平台完整操作手册

> 平台：https://agent-develop.rokid.com  
> 目标：零代码，在 Rokid Glasses 上实现语音刷题讲解智能体  
> 预计耗时：30～60 分钟

---

## 总体流程概览

```
第一步  注册 & 申请开发者权限
    ↓
第二步  创建知识库 → 上传题库 MD 文件
    ↓
第三步  创建智能体 → 填写基本信息
    ↓
第四步  配置系统提示词（Prompt）
    ↓
第五步  关联知识库
    ↓
第六步  配置工作流（可选，进阶）
    ↓
第七步  平台内调试测试
    ↓
第八步  手机 App 连接眼镜，部署上线
```

## 第一步：注册 & 申请开发者权限

### 1-1 注册 Rokid 账号

1. 打开手机，下载 **Rokid AI App**（App Store / 各大安卓应用商店搜索）
2. 注册账号，绑定手机号
3. ⚠️ 这个账号就是你后续所有操作的主账号，必须和灵珠平台一致

### 1-2 申请灵珠开发者权限

1. 在电脑浏览器打开：**https://agent-develop.rokid.com**
2. 点击右上角「**登录**」，选择手机号登录（和 App 同账号）
3. 首次进入会弹出「**申请开发者**」页面

   填写以下信息：
   | 字段 | 填写建议 |
   |------|----------|
   | 姓名 | 真实姓名 |
   | 职业 | 个人开发者 |
   | 开发方向 | AI 教育助手 / 学习工具 |
   | 开发目的 | 开发 LeetCode 刷题语音助手，帮助用户在佩戴眼镜时学习算法 |

4. 提交后等待审核，通常当天审核通过
5. 审核通过后会收到短信通知，重新登录平台即可进入创作中心

## 第二步：创建知识库

> 知识库是智能体的"大脑存储"，把题库文件上传进去，智能体才能回答具体题目

### 2-1 进入知识库页面

1. 登录灵珠平台后，左侧导航栏找到「**知识库**」
2. 点击右上角「**新建知识库**」按钮

### 2-2 填写知识库信息

| 字段 | 填写内容 |
|------|----------|
| 知识库名称 | LeetCode Hot100 题解库 |
| 描述 | 包含 LeetCode Hot100 全部 100 道经典算法题的题目、思路和 Python 代码，按哈希、双指针、链表等 14 大类型分类整理 |

点击「**确认创建**」

### 2-3 上传题库文件

1. 进入刚创建的知识库详情页
2. 点击「**上传文档**」或「**添加文件**」按钮
3. 选择本地文件：**`LeetCode_Hot100_题解库.md`**
   - 文件路径：`/Users/你的用户名/Desktop/20260316_rokid_智能体商店/LeetCode_Hot100_智能体/LeetCode_Hot100_题解库.md`
4. 等待系统解析（通常 1～3 分钟，文件较大时稍长）
5. 解析完成后，右侧可预览分段内容

### 2-4 配置分段规则（重要）

上传完成后，点击文件右侧「**设置**」或「**分段配置**」：

| 设置项 | 推荐值 | 说明 |
|--------|--------|------|
| 分段方式 | 自定义分隔符 | 按每道题之间的分隔线切分 |
| 分隔符 | `---` | 文件中每道题之间用三条横线分隔 |
| 最大分段长度 | 800 字符 | 避免一道题被截断 |
| 重叠长度 | 100 字符 | 保留上下文连接 |

点击「**保存并重新分段**」

### 2-5 验证分段结果

分段完成后，在知识库页面可以看到文档被拆成了多个片段：
- 正确的话，每个片段对应一道题（题号 + 题目 + 思路 + 代码）
- 如果发现题目被截断，需要调整「最大分段长度」到 1200 重试

## 第四步：配置系统提示词（核心步骤）

> 这是最关键的步骤，决定智能体的性格和回答质量

### 4-1 找到 Prompt 输入框

进入智能体编辑页后，找到：
- 「**角色设定**」/ 「**系统提示词**」/ 「**Persona & Prompt**」

（不同版本灵珠平台位置可能略有不同，通常在左侧或中间区域）

### 4-2 粘贴提示词

将以下内容**完整复制**粘贴进去：

```
你是一位 LeetCode 算法面试专家，专门帮助用户用语音学习 LeetCode Hot100 题目。

【角色设定】
你熟悉 LeetCode Hot100 全部 100 道经典算法题，涵盖哈希、双指针、滑动窗口、链表、二叉树、动态规划等 14 大类型，每道题都能用 Python 解答。

【核心交互规则】
用户会通过语音告诉你题号（如"第15题"）、题名（如"三数之和"）或类型（如"给我一道动态规划"）。

回答顺序：
1. 先用 1-2 句话说清楚题目是什么
2. 再说核心思路（不超过 3 句话）
3. 不要主动给代码，除非用户明确说"给我看代码"或"代码怎么写"
4. 用户追问时才展开细节

【语音适配要求】
- 每次回复控制在 80 字以内，适配眼镜语音播报
- 不使用任何表情符号、Markdown 格式、列表符号
- 语言自然口语化，像在和人讲话一样

【追问支持】
用户可以追问：
- "为什么用这个方法" → 解释算法选择原因
- "时间复杂度是多少" → 给出复杂度分析
- "给我看代码" → 给出 Python 完整代码
- "有没有更好的解法" → 对比其他解法
- "举个例子" → 用具体例子解释

【开场白】
"你好，我是你的 LeetCode Hot100 刷题搭档。告诉我题号或题目名称，我来帮你讲解思路。有不懂的随时追问。"
```

### 4-3 配置模型参数

在模型设置区域：

| 参数 | 推荐值 | 原因 |
|------|--------|------|
| 模型 | 平台默认（推荐 Qwen-Max 或 DeepSeek） | 代码理解能力强 |
| Temperature（温度） | 0.3～0.5 | 算法讲解需要准确，不要太随机 |
| 最大输出 Token | 512 | 语音播报不需要太长 |
| 上下文轮数 | 10 轮 | 保留追问的上下文 |

### 4-4 配置开场白

找到「**开场白**」或「**欢迎语**」输入框，填写：

```
你好！我是你的 LeetCode Hot100 刷题搭档。告诉我题号或题目名称，我来讲解解题思路。有不懂的地方随时追问，我会深入解释。
```

### 4-5 配置快捷指令（推荐）

找到「**快捷指令**」或「**预设问题**」区域，添加：

| 指令名称 | 指令内容 |
|----------|----------|
| 随机出一题 | 随机给我出一道 Hot100 题目，从简单题开始 |
| 动态规划 | 给我讲一道动态规划的经典题目 |
| 链表题 | 给我讲一道链表相关的题目 |
| 困难题挑战 | 给我出一道困难级别的题目 |
| 复杂度分析 | 帮我分析上道题的时间和空间复杂度 |

## 第五步：关联知识库

### 5-1 添加知识库

在智能体编辑页，找到「**知识库**」区域：
1. 点击「**添加知识库**」或「**+ 关联知识库**」
2. 从列表中选择：**LeetCode Hot100 题解库**
3. 点击「**确认**」

### 5-2 配置检索参数

添加知识库后，点击该知识库右侧「**设置**」：

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 检索策略 | 混合检索（语义 + 关键词） | 能同时处理题号查询和语义查询 |
| Top K（返回片段数） | 3 | 每次检索返回最相关的 3 个片段 |
| 相似度阈值 | 0.4 | 较低阈值保证召回率，避免找不到题目 |
| 最大 Token | 2000 | 知识库内容传给大模型的最大长度 |

### 5-3 在 Prompt 中引用知识库

在系统提示词末尾追加以下内容（让 AI 知道如何使用知识库）：

```
【知识库使用说明】
你有一个包含 LeetCode Hot100 题目的知识库。
当用户询问某道具体题目时，先从知识库检索该题的题目描述、核心思路和代码。
如果知识库中找到了相关内容，以知识库内容为准进行讲解。
如果知识库中没有找到，根据你的知识回答，并告知用户"这道题我从记忆中回答"。
```

## 第六步：搭建工作流（进阶，可选）

> 如果只是简单问答，前五步已经足够。工作流适合需要更复杂逻辑的场景。

### 6-1 为什么需要工作流

对于 LeetCode 智能体，工作流能实现：
- **意图识别**：区分用户是问题号、问题名、问题类型还是追问
- **检索优化**：根据不同意图走不同的检索策略
- **结构化输出**：按"题目→思路→追问引导"的固定格式回复

### 6-2 创建工作流

1. 左侧导航找到「**工作流**」
2. 点击「**新建工作流**」
3. 命名：`LeetCode 解题流程`
4. 进入可视化画布

### 6-3 搭建节点（拖拽操作）

按以下顺序从左侧节点库拖入画布并连线：

```
[开始节点]
    ↓
[大模型节点 - 意图识别]
    ↓
[条件判断节点]
    ↙        ↓        ↘
[知识库检索] [直接回答] [随机出题]
    ↓            ↓         ↓
        [大模型节点 - 生成回答]
                ↓
            [结束节点]
```

### 6-4 配置各节点

**① 开始节点**
- 输入变量名：`user_input`
- 类型：字符串
- 说明：用户的语音输入文字

**② 意图识别节点（大模型节点）**

系统提示词填写：
```
根据用户输入，判断用户意图，只输出以下之一：
- "specific_problem"：用户问了具体的题目（包含题号或题名）
- "random_problem"：用户想随机获取一道题
- "followup"：用户在追问上一道题的细节
- "category"：用户问某个类型的题
```

输入：`{{user_input}}`  
输出变量：`intent`

**③ 条件判断节点**

条件 1：`intent == "specific_problem"` → 走知识库检索分支  
条件 2：`intent == "random_problem"` → 走随机出题分支  
条件 3：其他 → 走大模型直接回答分支

**④ 知识库检索节点**

- 关联知识库：LeetCode Hot100 题解库
- 检索词：`{{user_input}}`
- Top K：3
- 输出变量：`retrieved_content`

**⑤ 生成回答节点（大模型节点）**

系统提示词：
```
你是 LeetCode 解题专家。
参考以下知识库内容回答用户问题，回复控制在 80 字内，适配语音播报。
知识库内容：{{retrieved_content}}
```
输入：`{{user_input}}`

**⑥ 结束节点**
- 输出：大模型节点的回答内容

### 6-5 调试工作流

1. 点击右上角「**测试**」按钮
2. 在输入框输入：`讲一下第15题`
3. 观察每个节点的输入输出是否正确
4. 如有错误，点击对应节点查看日志调整
5. 测试通过后，点击「**发布工作流**」

### 6-6 在智能体中关联工作流

回到智能体编辑页：
1. 找到「**工作流**」或「**插件**」区域
2. 点击「**添加工作流**」
3. 选择刚才创建的「LeetCode 解题流程」
4. 配置触发条件：「用户每次提问时触发」

## 第七步：平台内调试测试

### 7-1 打开调试面板

在智能体编辑页右侧，找到「**调试对话**」或「**预览**」面板

### 7-2 按以下顺序测试

**测试 1 - 基础问题**
```
输入：讲一下第1题
期望：说清楚两数之和是什么，讲哈希表思路，不主动给代码
```

**测试 2 - 追问代码**
```
输入：给我看代码
期望：给出 Python 代码
```

**测试 3 - 追问原理**
```
输入：为什么要用哈希表
期望：解释哈希表相比暴力枚举的优势
```

**测试 4 - 复杂度追问**
```
输入：时间复杂度是多少
期望：O(n)，并说明原因
```

**测试 5 - 按类型提问**
```
输入：给我一道动态规划的题
期望：推荐一道 DP 题并给出题目名和简要描述
```

**测试 6 - 困难题**
```
输入：接雨水怎么解
期望：讲解双指针或单调栈思路
```

### 7-3 常见问题处理

| 问题现象 | 原因 | 解决方法 |
|---------|------|----------|
| 回复太长，不适合语音 | Prompt 没限制字数 | 在 Prompt 里加：「每次回复严格控制在 80 字以内，这是硬性要求」 |
| 找不到具体题目 | 知识库未检索到 | 降低相似度阈值至 0.3，或增大 Top K 至 5 |
| AI 直接给代码没讲思路 | Prompt 顺序理解偏差 | 在 Prompt 里强调：「必须先说思路，代码只在用户明确要求时才给」 |
| 题目信息不准确 | 依赖 AI 记忆而非知识库 | 检查知识库关联是否正确，分段是否完整 |
| 追问时失去上下文 | 上下文轮数设置太少 | 将「上下文轮数」调整为 10～20 |

## 第八步：连接 Rokid Glasses 上线

### 8-1 保存智能体

在编辑页点击右上角「**保存**」，确认保存成功（页面显示"已保存"字样）

### 8-2 ⚠️ 不要点提审

- **未提审的智能体**：只有你自己的账号可以看到和使用（适合个人使用）
- **提审后的智能体**：会进入公开审核，所有用户可搜索到
- 对于个人刷题工具，**不需要提审，直接私用即可**

### 8-3 手机 App 端操作

1. 打开手机「**Rokid AI App**」
2. 确保手机账号与灵珠平台账号完全一致
3. 进入路径：**设置 → 开发者 → 智能体调试**
   
   > 如果没有看到「开发者」选项，说明开发者权限还未激活，需要等审核通过

4. 在智能体列表中，找到「**LeetCode 解题师**」
5. 点击右侧「**进入**」按钮

### 8-4 连接眼镜测试

1. 确保 Rokid Glasses 已开机并与手机蓝牙连接
2. 在手机 App 的智能体调试页面点击「进入」
3. 戴上眼镜
4. 说出唤醒词，然后说：「讲一下第53题」
5. 眼镜会语音播报：「最大子数组和。给定整数数组，找出连续子数组的最大和。核心思路是 Kadane 算法...」

### 8-5 配置快捷唤醒词

在 Rokid AI App 中：
1. 进入「**AI 快捷指令**」
2. 点击「**添加**」
3. 填写：
   - 唤醒词：`刷题` 或 `算法题`
   - 对应智能体：LeetCode 解题师
4. 保存后，直接对眼镜说「刷题」即可启动

## 第九步：持续优化建议

### 优化方向 1：分难度推荐

在 Prompt 里增加：
```
当用户说"随机给我一道题"时，先询问难度：简单、中等、还是困难？
然后从对应难度的题目中选一道推荐。
```

### 优化方向 2：学习进度记录

在工作流中添加「**变量存储节点**」：
- 每次用户听完一道题，记录题号到已学列表
- 用户说「今天学了哪些」时，列出已学题目

### 优化方向 3：错题追问

增加逻辑：
```
当用户说"我没听懂"时，换一种更简单的方式重新解释，并举一个具体的数字例子。
```

### 优化方向 4：考考我模式

增加交互玩法：
```
用户说"考考我"时，只说题目名，不给思路，等用户说出自己的解法，再评价对不对。
```
