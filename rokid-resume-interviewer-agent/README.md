# 🎤 简历面试官 · Rokid 灵珠智能体

> 基于 Rokid AR 眼镜的 AI 模拟面试智能体，读取用户简历或自我介绍后，全程针对性追问，还原真实大厂面试体验。

## 项目简介

不同于固定题库式的刷题练习，本智能体的核心逻辑是：**AI 只问你说过的东西**。

用户可以通过两种方式开始：
1. **直接说自我介绍**：AI 从你说的内容中提取项目、技术、实习经历，逐一深挖
2. **粘贴简历文字**：AI 读完简历后，针对每个项目和技能点进行针对性追问

每个人的面试体验完全不同，真正模拟"一对一面试官"的感觉。

## 功能特性

- 🎯 **100% 个性化**：完全基于用户提供的内容提问，不出固定题库
- 📄 **双入口设计**：自我介绍语音输入 + 简历文字粘贴，两种方式灵活切换
- 🔍 **深度追问**：同一话题最多追问 3 轮，从"是什么"追到"为什么"再到"底层原理"
- 📊 **实时评价**：随时说"评价我"获取当前表现反馈
- 🗣️ **语音优化**：每次回复 80 字以内，适配 Rokid 眼镜 TTS 语音播报
- 🚫 **不越界**：严格规则，AI 绝不问简历上没出现过的技术

## 快速开始

### 使用方式（灵珠平台）

1. 登录 [灵珠开发者平台](https://agent-develop.rokid.com)
2. 新建对话型智能体，名称填"简历面试官"
3. 将 `Prompt_系统提示词.md` 中「✅ 正式 Prompt」内的内容复制到系统提示词
4. 设置对话记忆轮数为 **20 轮**（关键！）
5. 发布，在 Rokid AI App 中启用

### 路径一：语音自我介绍

```
[戴上眼镜，唤醒 AI]
用户：你好，我要面试练习

AI：你好，你先做个自我介绍，说说专业、项目或实习经历，随意说就好，我听完再问你。

用户：我叫小明，本科计算机，做过一个电商后台项目，Spring Boot + MySQL，还用了 Redis 做缓存，然后有一段腾讯的实习，做推荐系统...

AI：你在腾讯实习做推荐系统，主要负责哪一块？

用户：我做的是特征工程，处理用户行为数据。

AI：特征工程里遇到过什么难搞的问题吗？

用户：主要是数据量很大，处理很慢。

AI：你是怎么解决数据处理慢这个问题的？
```

### 路径二：粘贴简历文字

```
[手机 App 中粘贴简历文字]
用户：这是我的简历：
姓名：李华，武汉大学软件工程2025届
技能：Java、Spring Boot、MySQL、Redis、Docker
项目：在线教育平台（2024.03-2024.06）
  - 引入 Redis 缓存，接口从 800ms 降至 120ms
实习：2024.07 美团，后端开发实习生

[戴眼镜，切换语音对话]
AI：好，我看完你的简历了。你做教育平台的时候，Redis 缓存把响应从 800ms 降到 120ms，这个数据是怎么测出来的？

用户：我用 Postman 对比了优化前后的接口响应时间。

AI：你加 Redis 之前，有没有先分析过为什么慢？

用户：有，先用 EXPLAIN 看了 MySQL 执行计划，发现是全表扫描，加索引还是慢，才上的缓存。

AI：思路不错。那你有没有考虑过加缓存之后数据一致性的问题？
```

## Prompt 设计亮点

| 特性 | 说明 |
|------|------|
| 最高优先规则 | 只问简历有的，绝不超纲，避免问到用户完全陌生的技术 |
| 每次一个问题 | 等用户回答后再追问，还原真实面试节奏 |
| 三轮追问上限 | 同一话题最多 3 轮，防止单点过度深挖 |
| 追问四优先级 | 为什么→难点→重做改哪→底层原理，层层递进 |
| 20 轮记忆 | AI 全程记得用户开头说的简历内容，实现连贯面试 |
