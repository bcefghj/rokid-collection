# 简历面试官 · 灵珠平台操作手册

> 平台：https://agent-develop.rokid.com  
> 核心玩法：用户说自我介绍 或 粘贴简历文字 → AI 面试官完全根据你说的内容追问  
> 预计搭建时间：20～30 分钟（比题库版更简单，不需要上传知识库）

---

## 和上一版的核心区别

| 对比项 | 校招面试官（题库版） | 简历面试官（本版） |
|--------|-----------------|----------------|
| 问题来源 | 固定题库（35道） | 完全来自用户说的内容 |
| 个性化程度 | 低（所有人问一样的） | 高（每人面试完全不同） |
| 是否需要知识库 | 需要上传MD文件 | 不需要，Prompt即核心 |
| 适合场景 | 系统性刷题练习 | 模拟真实面试，查漏补缺 |
| 搭建复杂度 | 中 | 低（只需配置Prompt） |

---

## 总体流程

```
第一步  注册 & 申请开发者权限（已完成可跳过）
    ↓
第二步  创建智能体 → 填写基本信息 & 上传图标
    ↓
第三步  配置系统提示词（Prompt）← 最核心的一步
    ↓
第四步  调试测试（重点测试两种使用路径）
    ↓
第五步  手机 App 连接眼镜，部署上线
```

> 注意：本版不需要创建知识库，Prompt 本身就是智能体的全部逻辑。

---

## 第一步：注册 & 申请开发者权限

如果已经在做 LeetCode 或海龟汤智能体时注册过，直接跳到第二步。

1. 下载 **Rokid AI App**，注册账号
2. 电脑打开 **https://agent-develop.rokid.com**，用同一账号登录
3. 申请开发者权限，填写开发目的：

   > "开发一个基于用户简历的 AI 模拟面试智能体，读取用户简历内容后进行针对性追问，帮助应届生练习真实面试场景"

---

## 第二步：创建智能体

### 2-1 新建智能体

1. 登录灵珠平台，左侧导航栏点击「**智能体**」
2. 点击「**新建智能体**」
3. 类型选择「**对话型智能体**」

### 2-2 填写基本信息

| 字段 | 填写内容 |
|------|----------|
| 智能体名称 | 简历面试官 |
| 简介 | 把你的简历文字发给我，或者直接说自我介绍，我来当你的面试官，专门追问你简历上的每一个细节，帮你找出准备不足的地方 |
| 分类 | 教育学习 / 职业发展 |
| 语言 | 中文 |

### 2-3 上传图标

点击头像区域「**上传图标**」，上传本次生成的图标。

图标描述：深蓝背景，中央悬浮发光简历文档，上方问号对话框，底部"简历面试官"白字，右上角金色眼镜图标。

---

## 第三步：配置系统提示词（最核心）

### 3-1 打开 Prompt 配置区

在智能体编辑页，找到「**系统提示词**」输入框（部分版本叫"角色设定"或"人物设定"）

### 3-2 复制粘贴 Prompt

打开 `Prompt_系统提示词.md`，将「✅ 正式 Prompt」下方三个反引号内的**全部文字**复制，粘贴到系统提示词输入框。

粘贴后检查以下关键内容是否完整：
- 【最重要的规则】那段（只问简历上有的）
- 【两种开始方式】（方式一自我介绍、方式二简历文字）
- 【提问策略】（每次只问一个问题）
- 【特殊指令响应】（换题、评价、结束）
- 【开场白】两个版本

### 3-3 模型参数配置

| 参数 | 推荐值 | 原因 |
|------|--------|------|
| 模型 | 平台默认推荐 | 无需特殊选择 |
| 温度（Temperature） | 0.6 | 面试场景需要稳定，不要太随机 |
| 最大回复长度 | 150 Token | 控制在80字以内，避免语音太长 |
| 对话记忆轮数 | **20 轮** | 关键！面试需要记住用户之前说过的所有内容 |

> 记忆轮数设置 20 轮非常重要。AI 面试官需要记住用户开头说的自我介绍或简历内容，才能在第 10 轮追问时还能引用"你之前提到的XX项目"。

### 3-4 不需要关联知识库

本版智能体无需知识库，AI 的"知识"就是用户自己提供的简历和自我介绍内容，全部存在对话上下文中。

## 第四步：调试测试

### 4-1 测试路径一：自我介绍版

在平台调试窗口，输入以下内容测试：

**测试输入：**
```
我叫小明，华中科大软件工程大四，做过一个仿抖音的短视频平台，后端用 Spring Boot，视频存 MinIO，推荐算法是基于协同过滤的，还有一段腾讯的实习，做过微信小程序的后台接口开发
```

**期望 AI 回复（示例）：**
```
你在腾讯实习做微信小程序后台，主要负责哪些接口？是增删改查类的，还是有复杂业务逻辑？
```

**验证点：**
- AI 是否只问了用户提到的内容（腾讯、MinIO、协同过滤）
- AI 是否一次只问了一个问题
- 回复是否在 80 字以内

### 4-2 测试路径二：简历文字版

**测试输入：**
```
这是我的简历：
技能：Python、Django、PostgreSQL、Redis、Linux
项目：图书馆座位预约系统（2024.09-2025.01）
  - 独立开发，前后端分离，后端 Django REST Framework
  - 用 Redis 实现座位实时状态更新，支持 500 并发
实习：2024.07 上海某创业公司，全栈开发实习，参与了用户管理模块开发
```

**期望 AI 回复（示例）：**
```
好，我看完你的简历了。你做了图书馆预约系统，支持 500 并发，这个数字是怎么得出来的？是压测过吗？
```

**验证点：**
- AI 是否先说了"我看完简历了"再开始提问
- AI 第一个问题是否来自简历里的具体内容
- AI 是否引用了用户简历里的实际项目名和数据

### 4-3 测试特殊指令

| 测试输入 | 期望行为 |
|---------|---------|
| `换一个话题` | AI 跳到简历上另一个点（如从项目跳到实习） |
| `评价我` | AI 给出1个具体亮点 + 1个改进建议，内容来自刚才的对话 |
| `结束` | AI 总结2个印象深刻的亮点 + 1个建议方向 |
| `我不知道` | AI 不直接给答案，给一个方向提示 |

### 4-4 常见问题排查

**问题：AI 问了简历上没有的技术**
- 原因：Prompt 里的规则没生效，或对话太长导致规则被遗忘
- 解决：在 Prompt 最开头加一行 `你只能根据用户提供的简历内容提问，这是最高优先级规则`

**问题：AI 一次问了两三个问题**
- 原因：模型倾向于详细回复
- 解决：在 Prompt 里加 `每次回复只能包含一个问题，这是硬性要求，不可违反`

**问题：AI 记不住用户开头说的简历内容**
- 原因：对话记忆轮数太少
- 解决：将记忆轮数调高到 20 轮，或在系统提示词里加 `用户最开始提供的简历内容是你的核心依据，请在整个面试过程中始终参考它`

**问题：回复有 Markdown 格式符号**
- 解决：在 Prompt 末尾加 `任何情况下都不能使用星号、横线、编号列表等格式符号`

## 第五步：手机 App 连接眼镜，部署上线

### 5-1 发布智能体

1. 调试满意后，点击右上角「**发布**」
2. 选择范围：「**仅自己**」（测试阶段）

### 5-2 两种使用方式详解

#### 方式一：语音自我介绍（眼镜直接用）

适合随时随地练习，通勤路上、睡前都可以。

```
操作步骤：
1. 戴上 Rokid 眼镜
2. 说唤醒词"小奇小奇"（或按侧键）
3. 切换到"简历面试官"

对话示例：
你说：你好，我要开始面试练习
AI说：你好，你先做个自我介绍，说说专业、项目或实习经历，随意说就好，我听完再问你
你说：我叫李华，本科计算机，做过一个外卖配送系统，用了 Go 语言写的...
AI说：你用 Go 写外卖系统，并发这块怎么处理的？用了 goroutine 还是有其他方案？
```

#### 方式二：先在手机粘贴简历，再戴眼镜语音对话

适合针对自己真实简历做专项练习。

```
操作步骤：
1. 打开手机 Rokid AI App
2. 进入"简历面试官"对话界面
3. 复制自己简历的文字内容，粘贴到输入框发送
4. AI 回复"好，我看完你的简历了，我们开始..."
5. 戴上眼镜，切换为语音对话，继续面试
```

> 技巧：简历文字不需要排版完美，把 Word 简历里的文字直接复制粘贴过来就行，AI 能识别。

### 5-3 实用使用技巧

**技巧一：每次面试前更新简历**
如果你的简历更新了，重新开一个对话，把新简历粘进去，AI 会基于最新版本来问。

**技巧二：聚焦一个模块练习**
如果只想练"项目经历"这一块，可以只粘贴简历中项目部分的文字，让 AI 集中追问这一块。

**技巧三：用"评价我"检验准备程度**
每次回答完几个问题后说"评价我"，AI 会指出你哪里说得不够清晰，帮你找盲区。

**技巧四：用"结束"获取总结报告**
说"结束面试"后，AI 会给出本次面试的整体评价，记录下来对比每次练习的进步。
